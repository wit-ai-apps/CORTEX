## 要約（CTX-001）

### ファイル名から `_saved-by_` を外した理由

- 保存者は **ヘッダの `SAVED_BY`** で示すのが正式。  
- ファイル名が長くなり、一覧・運用のノイズになる。  
- **日時先頭**命名と組み合わせて、一覧＝概ね時系列にそろえる方針と両立しやすい。

### 時系列（日付・時刻先頭）命名に変えた理由

- GitHub 等の **ファイル一覧が名前順＝概ね時系列** になり、受け渡しの追跡がしやすい。  
- 旧形式（`FROM_to_TO_日付_...`）は **レガシー** として残りうるが **新規は使わない**。

### `SOURCE` / `SAVED_BY` をヘッダ必須にした理由

- **誰が言ったか**（`FROM`）と **実生成元**（`SOURCE`）、**誰がリポジトリに保存したか**（`SAVED_BY`）を分ける。  
- 代理保存やチャット貼付などでも取り違えを減らす。

### 過去ファイルの扱い（固定方針）

- 既存の旧形式・旧ファイル名は **一括で巻き戻さない**。  
- 触るタイミングで **新形式へリネーム**するなど **段階的** に揃える。  
- **discussion 全文を Context にコピーしない**（要約だけを `context` に載せる）。
